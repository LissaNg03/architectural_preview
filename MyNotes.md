//THE LOGIC OF MY API::

//MY ADMIN:

1. CREATE AN UPLOAD FIELDS VARIABLE WICH IS AN ARRAY OF OBJECTS THAT DEFINE THE FIELDS WITH FEILD NAMES AND MAXCOUNT OF THE FILES OF EACH FIELD, FIELD NAMES MUST BE THE SAME AS THE FIELD NAMES SENT FROM THE FRONTEND TO MATCH EACH FILE

OBJECT DESCTRUCTURE THE FILES FROM THE REQ.FILES

2. I GET THE EXISTING ADMIN DATA FROM THE DATABASE, AND RETRIEVE THE FILES METADATA, THE FILES ARE BUSINESS LOGO, HERO IMG, AND WHO WE ARE IMG, THESE ACTUALLY CONTAIN THE THE IMG CLOUDINARY URL, AND PUBLIC ID, ASSIGN THIS TO A VARIABLE "existing" IT'LL RETURN NULL IF THERE'S NO EXISTING ADMIN DATA

3. business_logo, hero_img, who_we_are_img are array destructured assigned to return promises using the await Promise.all() where foreach of these, the resolveImg function is called and given (field_name of the image, the image url, and public id, these are retrieved from the "existing" variable)

4. THE RESOLVEIMAGE() FUNCTION IS AN ASYNCHRONOUS FUNCTION HENCE WE USED PROMISE.ALL() FOREACH OF THE CALLS, IT ACCEPTS (fieldName, img.url, existingPublicId = null), using the fieldname, it checks if the fieldname is sent from the frontend, i.e req.files|fieldname|, if it's sent, then it uploads to cloudinary by calling the uploadToCloudinary() method passing the file.buffer, and public_id....if the fieldname file is not sent from the frontend, it returns the existing img.url and public_id...THIS IS DONE SO THIS METHOD RETURNS THE UPDATED VERSION OF THE IMG.URL AND PUBLIC ID TO THE VARIBLES DESTRUCTURED

5. THE UPLOADTOCLOUDINARY() METHOD IS CALLED BY THE RESOLVEIMG() METHOD IF THE FIELDNAME PASSED FILE IS SENT FROM FRONTEND, THIS METHOD RETURNS A NEW PROMISE TO THE RESOLVEINMG() METHOD SINCE IT ASYNCHRONOUSLY AWAITS CALLING IT, SO THIS METHOD RETURNS A PROMISE() WITH RESOLVE,REJECT, ALL IT DOES IS UPLOAD THE FILE BUFFER TO CLOUDINARY, IT ACCEPTS(FILE.BUFFER, PUBLIC_ID) IF PUBLIC ID EXISTS, COZ NOT ALWAYS THE PUBLIC ID EXIST SINCE WHEN THE ADMIN DATA FOR FIRST TIME, NO DATA EXISTS YET, SO THIS UPLOADS TO CLOUDINARY USING THE cloudinary.uploader.upload_stream() WHERE IT {METADATA LIKE RESOURCE_TYPE, FOLDER, OVERWRITE, PUBLIC_ID,INVALIDATE},(ERR,RESULT) => {RESOLVE,REJECT}, THEN UPLOADER.END(FILE.BUFFER)

6. THEN WE UPDATE THE DATABASE WITH FINDONEANDUPDATE({FILTER, singletonKey: "ADMIN_CONFIG" },{DATA},{OPTIONS, upsert: true,returnDocument:"after"})
