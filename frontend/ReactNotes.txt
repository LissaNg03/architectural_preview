MODULE:
MODULE styling helps avoid css naming conflicts, so module styling goes like this (e.g we are styling a button uniquely):

<button className = "btn" id="btn-id">Click Me<button/>

1. You create a Folder named Button, inside it is a Button.jsx comp, and a Button.module.css file, this is important so that the className of this component gets generated uniquely.
2. You import the style sheet 'Button.module.css' like you would import a Component i.e import styleName from './Button.module.css', can name it anything
3. set className = {styleName.btn}, if you referenced your Button with a class or id = {styleName.btn-id}

THEN YOUR STYLING WILL WORK WITH UNIQUE CLASS/ID THAT CAN NEVER BE CONFLICTED!

CLICK EVENTS IN REACT:

E.G onClick={callback} = An interaction when a user clicks on a specific element, We can respond clicks by passing a callBack.
* when you call a function on click event you must make sure you dont use brackets like callback(), this will call the function immediately without waiting for a click of a button,

* In case you have arguments to pass to the function, you can use this instead: onClick={() => callback(args)}, use a arrow function when calling the function onClick, onClick={() => callback(args)};

THERE ARE A LOT OF EVENT LISTENERS e.g oncClick, onDoubleClick etc, you can take advantage of the event object and a LOT OF DOM MANIPULATION

updater function = A function passed as an argument to useState() usually e.x setYear(y => y + 1)
        Allow for safe updates based on the previous state
        Used with multiple state upgrades and asynchronous functions 
        Good practice to use updater functions

Takes the PENDING state to calculate NEXT state.
React puts your updater function in queue (waiting in line)
During the next render, it will call them in the same order. 

NB: Use the updater function whenever the value you want is depended on the previous state

import React, {useState} from 'react'

import React, {useState} from 'react'

export default function UpdateObject(){

    const [car, setCar] = useState({year: 2024,
                                    make:"Mercedez Benz",
                                    model:"AMG CLE"
                                    })

    
    function handleYearChange(event){
        setCar(c => ({...c, year: event.target.value}))
    }                                
    
    function handleMakeChange(event){
        setCar(c => ({...c, make: event.target.value}))
    }                                
    
    function handleModelChange(event){
        setCar(c => ({...c, model: event.target.value}))
    }      
    
    return ( <div>
                <p>Your favourite car is: {car.year} {car.make} {car.model}</p>

                <input type="number" value={car.year} onChange={handleYearChange} ></input><br></br>

                <input type="text" value={car.make} onChange={handleMakeChange} ></input><br></br>

                <input type="text" value={car.model} onChange={handleModelChange} ></input><br></br>
            </div>)

}

### 🧠 React: `useState()` vs `useRef()` – Quick Notes

#### 🔁 Re-renders in React
Every time state or props change, React **re-runs the entire component function**, meaning:
- All **regular variables (`let`, `const`, `var`) are reset**.
- You lose any values stored in them — unless you use React hooks.

---

#### ✅ `useState()`

- **Purpose:** Stores values that should affect the UI and trigger a re-render.
- **Persists across re-renders.**
- **Triggers a re-render when updated.**

```js
const [count, setCount] = useState(0);
setCount(count + 1); // Triggers re-render with new value
```

---

#### ✅ `useRef()`

- **Purpose:** Stores values that should persist between re-renders **but don’t affect the UI**.
- **Persists across re-renders.**
- **Does NOT trigger a re-render when updated.**
- **Access with `.current`** like this: `myRef.current = 123;`

```js
const intervalId = useRef(null);
intervalId.current = setInterval(() => {}, 1000);
```

🧠 Think of `useRef()` like a **static variable** in vanilla JS or a **hidden notepad** that doesn’t disturb the UI.

---

### 📌 When to Use What

| Use Case                         | Hook to Use |
|----------------------------------|-------------|
| Track user input or counter      | `useState()`|
| Store timers, IDs, DOM refs      | `useRef()`  |
| You want to trigger re-renders   | `useState()`|
| You want to persist without re-render | `useRef()`  |

## 🔹 What is `useRef`?
`useRef` is a React hook that gives you a **mutable reference** object that **persists across renders**.

It has two main uses:
1. **Referencing DOM elements**
2. **Storing values that don’t trigger re-renders**

---

## ✅ 1. How to Use `useRef` to Store Values

```js
import React, { useRef, useState } from "react";

export default function RefExample() {
  const renderCount = useRef(0);
  const [text, setText] = useState("");

  const handleChange = (e) => {
    setText(e.target.value);
    renderCount.current += 1;
  };

  return (
    <div>
      <input value={text} onChange={handleChange} />
      <p>You've typed: {text}</p>
      <p>This component rendered: {renderCount.current} times</p>
    </div>
  );
}
```

🧠 `renderCount.current` will update on every change, but **it won’t trigger a re-render**, so it’s super efficient for tracking values behind the scenes.

---

## ✅ 2. Referencing a DOM Element (e.g., input focus)

```js
import React, { useRef } from "react";

export default function InputFocus() {
  const inputRef = useRef(null);

  const focusInput = () => {
    inputRef.current.focus();
  };

  return (
    <div>
      <input ref={inputRef} type="text" placeholder="Click the button to focus me" />
      <button onClick={focusInput}>Focus Input</button>
    </div>
  );
}
```

🧠 `ref={inputRef}` links the element to your reference, and then you can call methods like `inputRef.current.focus()`.

---

## ⚠️ Important Notes:

- `useRef()` returns an object like this:  
  ```js
  { current: initialValue }
  ```

- It doesn’t cause a re-render when `current` changes.
- Perfect for:
  - DOM manipulation
  - Timers / intervals
  - Storing previous values
  - Mutable data without re-render

---

## ✅ Summary Cheat Sheet

| Purpose               | Code Sample                              |
|------------------------|-------------------------------------------|
| Store mutable value    | `const counter = useRef(0)`               |
| Access DOM element     | `const inputRef = useRef(null)`           |
| Use DOM ref            | `<input ref={inputRef} />`               |
| Access ref value       | `inputRef.current.focus()`                |
| Keep value on render   | `myRef.current = updatedValue`           |

## 🔹 What is React Router?

React Router is a **library** that lets you create **multi-page experiences** in React apps without reloading the page. It does this using the **History API** in the browser.

Instead of the whole page reloading when you go from `/about` to `/contact`, React Router just swaps components in place depending on the current **URL path**.

---

## 🔹 Core Concepts

### 1. **Routes**

A `Route` maps a **URL path** to a **React component**.

```jsx
import { BrowserRouter, Routes, Route } from "react-router-dom";
import Home from "./Home";
import About from "./About";

export default function App() {
  return (
    <BrowserRouter>
      <Routes>
        <Route path="/" element={<Home />} /> 
        <Route path="/about" element={<About />} /> 
      </Routes>
    </BrowserRouter>
  );
}
```

* When URL is `/`, `<Home />` shows.
* When URL is `/about`, `<About />` shows.

---

### 2. **Navigation (`useNavigate`)**

Instead of `<a>` tags (which reload the page), React Router gives us `useNavigate`.

```jsx
import { useNavigate } from "react-router-dom";

function Home() {
  const navigate = useNavigate();

  return (
    <div>
      <h1>Home</h1>
      <button onClick={() => navigate("/about")}>Go to About</button>
    </div>
  );
}
```

➡ Clicking the button changes URL to `/about` **without refreshing**.

---

### 3. **Location (`useLocation`)**

`useLocation` gives info about the current URL (pathname, search params, state).

```jsx
import { useLocation } from "react-router-dom";

function About() {
  const location = useLocation();

  return (
    <div>
      <h1>About</h1>
      <p>Current path: {location.pathname}</p>
    </div>
  );
}
```

If URL is `/about?user=123`,
`location.search` → `"?user=123"`

---

### 4. **Nested Routes & Children**

You can nest routes inside each other with `children`.
Example: `/dashboard` and `/dashboard/profile`.

```jsx
import Dashboard from "./Dashboard";
import Profile from "./Profile";
import Settings from "./Settings";

<Routes>
  <Route path="/dashboard" element={<Dashboard />}>
    <Route path="profile" element={<Profile />} />
    <Route path="settings" element={<Settings />} />
  </Route>
</Routes>
```

Inside `Dashboard`, you use `<Outlet />` to render the child routes:

```jsx
import { Outlet } from "react-router-dom";

function Dashboard() {
  return (
    <div>
      <h1>Dashboard</h1>
      <Outlet /> {/* Profile or Settings will show here */}
    </div>
  );
}
```

---

### 5. **Layout Routes**

A Layout is just a component (like a template) that wraps other pages.

Example: A layout with a **navbar + footer** that appears on every page.

```jsx
function Layout() {
  return (
    <div>
      <nav>Navbar</nav>
      <Outlet /> {/* Child pages render here */}
      <footer>Footer</footer>
    </div>
  );
}

<Routes>
  <Route path="/" element={<Layout />}>
    <Route index element={<Home />} />  {/* default child */}
    <Route path="about" element={<About />} />
    <Route path="contact" element={<Contact />} />
  </Route>
</Routes>
```

Now all pages (`/`, `/about`, `/contact`) share the same **Layout**.

---

### 6. **Other Hooks**

* `useParams()` → Get route params (e.g., `/users/:id`)
* `useSearchParams()` → Manage query strings (`?sort=asc`)
* `useNavigationType()` → Detect if user navigated via back/forward button

---

✅ **In short:**

* `BrowserRouter` → wraps app, controls history.
* `Routes` → define mapping between paths and components.
* `Route` → one path-to-component rule.
* `useNavigate` → programmatic navigation.
* `useLocation` → info about current URL.
* `Outlet` → where child/nested routes render.
* Layout = reusable wrapper with header/footer etc.

⚡ Important Notes:

1. `state` is **not visible in the URL** (unlike query params).
2. If you refresh the page, the state is lost (because it’s stored in memory, not in the URL).
3. Use `query params` (`useSearchParams`) if you want the data to persist in the URL.
///////////////////////////////////////////////////
Exactly ✅ — that’s how `state` works in React Router:

* **`state` lives only in memory**.
* If you refresh the page, `location.state` becomes `undefined`.

So if your page **relies entirely on state**, you should **always check if it exists** and handle the missing case, like redirecting to `/`:

---

### Example pattern

```jsx
import { useLocation, useNavigate } from "react-router-dom";
import { useEffect } from "react";

export default function About() {
  const location = useLocation();
  const navigate = useNavigate();
  const { user, age } = location.state || {};

  // Redirect if state is missing (e.g., page refreshed)
  useEffect(() => {
    if (!user) {
      navigate("/",{state: {}}, { replace: true });//prevents the user from going back to the broken page with the back button.
    }
  }, [user, navigate]);

  if (!user) return null; // optional fallback while redirecting

  return (
    <div>
      <h1>About Page</h1>
      <p>User: {user}</p>
      <p>Age: {age}</p>
    </div>
  );
}
```

* `replace: true` → prevents the user from going back to the broken page with the back button.
* This way, your page never breaks if state is missing.

---

This is **the safe way to rely on `state`** without worrying about a refresh.

////////////////////////////////USING USEPARAMS/////////////////////////
Exactly ✅ but with one key difference: **how you pass it**. Let me clarify:

---

### 1️⃣ Using `state` (accessed via `useLocation`)

```jsx
navigate("/about", { state: { user: "Lisakhanya" } });
```

* Access on `/about` with:

```jsx
const { state } = useLocation();
console.log(state.user); // "Lisakhanya"
```

* **Not visible in URL**, lost on refresh.

---

### 2️⃣ Using query params (accessed via `useSearchParams`)

```jsx
navigate("/about?user=Lisakhanya");
```

or dynamically:

```jsx
navigate(`/about?user=${user}`);
```

* Access on `/about` with:

```jsx
const [searchParams] = useSearchParams();
console.log(searchParams.get("user")); // "Lisakhanya"
```

* **Visible in URL**, survives refresh.

---

### ✅ Key difference in passing:

| Method       | How you pass data                      | Access method         | Survives refresh? |
| ------------ | -------------------------------------- | --------------------- | ----------------- |
| `state`      | `navigate("/about", { state: {...} })` | `useLocation().state` | ❌                 |
| query params | `navigate("/about?user=abc")`          | `useSearchParams()`   | ✅                 |

---

💡 **Rule of thumb:**

* Use **state** for temporary in-memory data.
* Use **query params** for persistent data you might refresh, bookmark, or share in a URL.
